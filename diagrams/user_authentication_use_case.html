Diagram Description:

This use case diagram illustrates the authentication flows in the "What's It Like" application:

  • Standard Authentication: Users can register and login with email/password
  • Social Authentication: Users can authenticate using Google
  • ID Verification: A critical security step requiring users to verify their identity with a Bosnian ID card
  • Password Recovery: Users can reset their password if forgotten
  • Session Management: The system maintains user sessions after successful authentication

The diagram highlights the mandatory ID verification step, which is unique to this application and ensures users are physically located in Bosnia and Herzegovina.

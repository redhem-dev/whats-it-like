        E->>F: Production Environment Detected

            F->>U: Request Browser Geolocation Permission

                U->>F: Allow Location Access
                F->>G: Get Coordinates
                G->>F: Return Latitude/Longitude
                F->>G: Reverse Geocode Coordinates
                G->>F: Return Country Information
            else Permission Denied
                U->>F: Deny Location Access
                F->>F: Fall Back to IP-based Location
            end
        else Private/Local IP
            F->>F: Detect Private/Local IP
        end
        
        alt Using IP Fallback
            F->>I1: Request Location from Primary IP API
            
            alt API 1 Success
                I1->>F: Return Country Information
            else API 1 Failed/Rate Limited
                F->>I2: Request Location from Secondary IP API
                
                alt API 2 Success
                    I2->>F: Return Country Information
                else API 2 Failed
                    I2->>F: API Error
                    F->>U: Display Location Error
                    Note over U: Cannot Proceed Without Location
                end
            end
        end
        
        F->>L: Validate Country
        
        alt Country is Bosnia
            L->>F: Location Verified
            F->>U: Continue with Verified Bosnia Location
        else Country is Not Bosnia
            L->>F: Location Outside Bosnia
            F->>U: Display Location Restriction Error
            Note over U: Cannot Access Location-Restricted Feature
        end
    end

Location Verification Process

This sequence diagram illustrates the location verification process used to ensure users are accessing the platform from Bosnia:

  • Environment Detection: Different behavior for development vs. production environments
  • Geolocation Strategy: Primary method using browser geolocation with user permission
  • IP Fallback: Secondary method using IP-based location when browser geolocation is unavailable
  • Redundant APIs: Multiple IP location APIs for reliability
  • Validation: Final check to confirm the user is in Bosnia

The diagram shows the complete flow including all fallback mechanisms and error handling scenarios.

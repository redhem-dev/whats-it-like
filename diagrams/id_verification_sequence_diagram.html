ID Verification Sequence Diagram

sequenceDiagram
    autonumber
    participant U as User
    participant F as Frontend
    participant A as Auth Service
    participant O as OCR Service
    participant V as Verification Service
    participant D as Database

    Note over U,D: ID Verification Process

    %% Start verification
    U->>F: Initiate Registration
    alt Google Auth
        F->>A: Request Google Authentication
        A->>U: Redirect to Google Auth
        U->>A: Provide Google Credentials
        A->>F: Return Authentication Token
        F->>F: Store Session Data
    else Standard Auth
        F->>U: Show Registration Form
        U->>F: Submit Registration Details
        F->>A: Create Temporary Account
        A->>F: Registration Started
    end

    %% ID Upload
    F->>U: Display ID Upload Form
    U->>F: Upload ID Document
    F->>O: Process ID Image

    %% OCR Processing
    O->>O: Run OCR on Image
    O->>V: Send Extracted Text
    
    %% ID Validation
    V->>V: Detect ID Type
    alt Bosnian ID Detected
        V->>V: Extract ID Number
        V->>V: Extract Name
        V->>V: Validate ID Format
        
        alt Valid Format
            V->>D: Check if ID Already Registered
            
            alt ID Not Registered
                V->>A: Create User Account
                A->>A: Generate Avatar
                A->>D: Set Up User Profile
                D->>A: Profile Created
                A->>F: Verification Success
                F->>U: Display Success & Redirect
            else ID Already Registered
                D->>V: ID Already Exists
                V->>F: Verification Failed
                F->>U: Display Error (ID Already Used)
                Note over U: Retry with Different ID
            end
        else Invalid Format
            V->>F: Invalid ID Format
            F->>U: Display Error (Invalid Format)
            Note over U: Retry with Valid ID
        end
    else Non-Bosnian ID
        V->>F: Unsupported ID Type
        F->>U: Display Error (Unsupported ID)
        Note over U: Retry with Bosnian ID
    end

ID Verification Process

This sequence diagram illustrates the interaction between different components during the ID verification process:

  • Authentication Flow: Users can register via Google or standard authentication
  • ID Upload: User uploads their Bosnian ID for verification
  • OCR Processing: The system extracts information from the ID using OCR
  • Validation: The system validates the ID format and checks for duplicates
  • Account Creation: Upon successful verification, a user account is created

The diagram shows all possible paths including error scenarios and retry flows.

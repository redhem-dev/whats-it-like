Diagram Description:

This use case diagram illustrates the voting system in the "What's It Like" application:

  • Vote Actions: Users can upvote or downvote both posts and replies
  • Vote Removal: Users can remove their previous votes
  • Location Verification: All voting actions require verification that the user is physically located in Bosnia and Herzegovina
  • Reputation System: Votes trigger updates to user reputation scores
  • External Dependencies: The system relies on location APIs to verify user location

The diagram emphasizes the mandatory location verification for all voting actions, which is a core security feature ensuring that only users physically present in Bosnia and Herzegovina can influence content ranking.

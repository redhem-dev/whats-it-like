Diagram Description:

This system architecture diagram provides a high-level overview of the "What's It Like" application's components and their interactions:

  • Frontend Architecture:
    • React-based with component-driven design
    • Context API for state management (LocationContext, AuthContext)
    • Custom hooks for reusable logic
    • Service layer for API communication
  • Backend Architecture:
    • Node.js/Express server with RESTful API endpoints
    • Controller-based architecture for request handling
    • MongoDB models for data structure
    • Middleware for authentication and request processing
    • Service layer for business logic and external API integration
  • External Services:
    • Google OAuth for social authentication
    • Geolocation APIs (ipapi.co, ip-api.com) for location verification
    • OCR service for ID card verification
    • OpenStreetMap for reverse geocoding
  • Database: MongoDB collections for users, posts, replies, and verification data

The architecture emphasizes the separation of concerns between frontend and backend components, with clear interfaces between layers. The context providers play a central role in managing application state, particularly for authentication and location verification.

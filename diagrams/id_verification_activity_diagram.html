                stateDiagram-v2
                    [*] --> BeginRegistration
                    
                    BeginRegistration --> SelectAuthMethod
                    
                    state auth_choice <>
                    SelectAuthMethod --> auth_choice
                    auth_choice --> GoogleAuth: Google Login
                    auth_choice --> StandardAuth: Email & Password
                    
                    GoogleAuth --> ProvideIDDocument
                    StandardAuth --> ProvideIDDocument
                    
                    ProvideIDDocument --> VerifyDocument
                    
                    state document_valid <>
                    VerifyDocument --> document_valid: Check ID
                    document_valid --> IDVerified: Valid Bosnian ID
                    document_valid --> IDRejected: Invalid or Non-Bosnian ID
                    
                    state id_unique <>
                    IDVerified --> id_unique: Check if ID is unique
                    id_unique --> CreateAccount: ID Not Registered
                    id_unique --> IDRejected: ID Already Used
                    
                    CreateAccount --> CompleteRegistration
                    
                    CompleteRegistration --> [*]: Success
                    
                    IDRejected --> RetryVerification
                    RetryVerification --> ProvideIDDocument: Try Again
                    
                    RetryVerification --> CancelRegistration: Cancel
                    CancelRegistration --> [*]: Registration Aborted

Diagram Description:

This simplified activity diagram shows the core ID verification process for the "What's It Like" application:

  • Main Flow: Shows the primary path from registration to account creation
  • Key Decisions:
    • Authentication method choice (Google or Standard)
    • ID document validity check
    • Uniqueness check (ensuring ID hasn't been used before)
  • Core Process: Users must provide a valid Bosnian ID document to register
  • Outcomes: Either successful account creation or rejection with retry option

This high-level diagram focuses on the critical decision points in the verification flow rather than implementation details, showing how the system ensures only users with valid Bosnian IDs can create accounts.

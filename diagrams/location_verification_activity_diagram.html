                    [*] --> CheckEnvironment
                    
                    state if_dev <>
                    CheckEnvironment --> if_dev
                    if_dev --> BypassVerification: Development Mode
                    if_dev --> CheckIPType: Production Mode
                    
                    BypassVerification --> ReturnMockLocation
                    ReturnMockLocation --> [*]: Return Bosnia Location
                    
                    state if_private_ip <>
                    CheckIPType --> if_private_ip
                    if_private_ip --> TryBrowserGeolocation: Public IP
                    if_private_ip --> UseIPFallback: Private/Local IP
                    
                    TryBrowserGeolocation --> BrowserPermission: Request Location
                    
                    BrowserPermission --> GetCoordinates: Permission Granted
                    BrowserPermission --> UseIPFallback: Permission Denied
                    
                    GetCoordinates --> ReverseGeocode: Get Lat/Long
                    ReverseGeocode --> ValidateCountry: Get Country
                    
                    UseIPFallback --> CallIPAPI1: Try Primary API
                    CallIPAPI1 --> CheckIPAPI1Response
                    
                    CheckIPAPI1Response --> ValidateCountry: Success
                    CheckIPAPI1Response --> CallIPAPI2: Failed/Rate Limited
                    
                    CallIPAPI2 --> CheckIPAPI2Response
                    CheckIPAPI2Response --> ValidateCountry: Success
                    CheckIPAPI2Response --> LocationError: All APIs Failed
                    
                    state if_bosnia <>
                    ValidateCountry --> if_bosnia
                    if_bosnia --> LocationVerified: Country is Bosnia
                    if_bosnia --> LocationFailed: Not Bosnia
                    
                    LocationVerified --> CacheResult: Store in Context
                    CacheResult --> [*]: Return Success
                    
                    LocationFailed --> [*]: Return Failure
                    LocationError --> [*]: Return Error

Diagram Description:

This activity diagram illustrates the location verification process in the "What's It Like" application:

  • Environment Detection: The system first checks if it's running in development or production mode
  • Development Bypass: In development environments, location verification is bypassed with mock Bosnia location data
  • IP Type Detection: The system identifies if the user is on a private/local IP or public IP
  • Multiple Verification Methods:
    • Browser Geolocation API (preferred method)
    • IP-based geolocation using multiple fallback services
  • Country Validation: Verification that the user is physically in Bosnia and Herzegovina
  • Result Caching: Successful verifications are cached to avoid repeated checks

The diagram highlights the robust multi-layered approach to location verification, with development mode bypass for easier testing and multiple fallback mechanisms to ensure reliable verification in production.

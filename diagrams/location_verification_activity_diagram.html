                    [*] --> RequestLocation
                    
                    RequestLocation --> AttemptVerification
                    
                    state if_dev <>
                    AttemptVerification --> if_dev: Check Environment
                    if_dev --> LocationVerified: Development Mode
                    if_dev --> GetUserLocation: Production Mode
                    
                    state if_method <>
                    GetUserLocation --> if_method: Determine Method
                    if_method --> BrowserGeolocation: Try Browser First
                    if_method --> IPGeolocation: Fallback
                    
                    BrowserGeolocation --> ValidateLocation: Success
                    BrowserGeolocation --> IPGeolocation: Failed
                    
                    IPGeolocation --> ValidateLocation: IP Located
                    IPGeolocation --> LocationError: Unable to Locate
                    
                    state if_bosnia <>
                    ValidateLocation --> if_bosnia: Check Country
                    if_bosnia --> LocationVerified: Is Bosnia
                    if_bosnia --> LocationError: Not Bosnia
                    
                    LocationVerified --> [*]: Success
                    LocationError --> [*]: Failure

Diagram Description:

This simplified activity diagram illustrates the core location verification process in the "What's It Like" application:

  • Main Flow: Shows the primary path for verifying a user's location
  • Key Decisions:
    • Whether user is in development or production environment
    • Which geolocation method to use (browser or IP-based)
    • Whether the detected location is in Bosnia
  • Outcomes: Either successful verification or failure with appropriate error handling

This diagram focuses on the high-level workflow rather than implementation details, showing how the system determines whether a user is physically located in Bosnia before allowing location-restricted features.
